Model 1 — Single vs. Binary classifier Simple

Upload a dataset of light curves and let the trained classifier predict, for each event, whether it is a single-lens or binary-lens microlensing event.

What this model is

A 1D convolutional neural network (PyTorch) that reads the shape of a light curve and classifies it as single-lens or binary-lens. It learns to tell the smooth, symmetric Paczyński peak of a single lens apart from the sharp caustic-crossing spikes of a binary lens.

How it was created

Trained on 100,000 synthetic I(t) light curves generated above (90% single / 10% binary, 400 points each), with no noise or cadence gaps — the Simple framework. Each curve is normalized individually before training, and the 90/10 class imbalance is handled with a weighted loss. On a held-out test set it reaches F1 ≈ 0.999.

Before you upload: this model only accepts a model dataset — a file containing only the light-curve columns (t_000 … t_399) and nothing else, with exactly 400 points per curve and no missing values. Files that include extra parameter columns, a different number of points, or cadence gaps (NaN) will be rejected without being run through the model.

Model 1 — Single vs. Binary classifier Real

The realistic counterpart: it reads noisy, cadence-gapped, blended OGLE-like curves. Results are reported at two stages — a permissive general candidate list and a clean strict catalogue.

What this model is

A 1D CNN taking four channels: the magnitude curve, an observed mask, and the fold residual I(τ)−I(−τ) with its own mask. A single lens is symmetric in τ, so a binary's signature is exactly that departure from symmetry — the model is handed it directly.

Two stages, and why

Scores are calibrated, so a probability is real (0.66 means ≈66% of such events are binary) and a threshold is a precision target. General (P ≥ 0.5) is a permissive candidate list to hand on for review; strict is chosen for high precision — the clean catalogue.

Honest expectation: on a realistic planet-heavy population most binaries leave anomalies below the photometric noise, so they are physically undetectable. This model runs near that information ceiling (AUC ≈ 0.55): the strict stage buys precision by giving up completeness. Unlike the Simple model it accepts cadence gaps (NaN) as well as clean curves, but still needs only light-curve columns (t_000 … t_399), exactly 400 points.
